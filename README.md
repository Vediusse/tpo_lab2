# lab2-function-system

Полностью готовый Maven-проект на Java 17 для лабораторной работы по интеграционному тестированию системы функций. Проект можно собрать, протестировать, запустить как fat jar, получить CSV-выгрузки и построить по ним графики для отчета.

## 1. Что Это За Лабораторная Работа

Суть лабораторной в том, чтобы не просто написать вычисление системы функций, а показать осмысленное интеграционное тестирование модулей, из которых эта система собирается. Нужно:

- реализовать базовые функции численными методами;
- выразить производные функции только через базовые;
- собрать из модулей систему функций;
- поддержать табличные заглушки;
- экспортировать значения в CSV;
- написать unit tests и integration tests;
- подготовить данные для графиков и отчета.

## 2. Постановка Задачи

Для варианта требуется вычислять кусочную систему функций:

Для `x <= 0`:

```text
((((tan(x)^2 * tan(x) + cos(x)) * sec(x)) / (csc(x) * tan(x) - tan(x))) - cos(x))
```

Для `x > 0`:

```text
((((log_2(x)^3)^2 + log_2(x))^2) - (log_5(x) + (ln(x) * ((log_3(x)^2) + (log_2(x)^3)))))
```

## 3. Ограничения Из Задания

- Нельзя упрощать всю систему тригонометрическими или логарифмическими преобразованиями.
- Все функции должны быть построены через базовые.
- Базовая тригонометрическая функция: `sin(x)` через ряд Тейлора.
- Базовая логарифмическая функция: `ln(x)` через численный ряд.
- Должны быть табличные заглушки модулей.
- Нужно уметь делать CSV-экспорт значений функций.
- Для защиты нужны графики и объяснимое тестовое покрытие.

## 4. Что Реализовано В Проекте

- `sin(x)` через `TaylorSinCalculator`
- `ln(x)` через `LnSeriesCalculator`
- производные тригонометрические функции:
  - `cos`
  - `tan`
  - `cot`
  - `sec`
  - `csc`
- производные логарифмические функции:
  - `log2`
  - `log3`
  - `log5`
- кусочная система функций `FunctionSystem`
- табличные заглушки `TableFunction`
- сэмплирование значений `FunctionSampler`
- экспорт в CSV `CsvExporter`
- фабрика production-конфигурации `FunctionFactory`
- запускаемое приложение `Application`
- Python-скрипт для построения графиков

## 6. Объяснение Пакетов Человеческим Языком

### `approximation`

Здесь лежит математика нижнего уровня:

- `TaylorSinCalculator` считает `sin(x)` через ряд Тейлора;
- `LnSeriesCalculator` считает `ln(x)` через численный ряд.

Это базовые кирпичики, от которых зависит остальная система.

### `function.trig`

Здесь каждая функция построена через базовую тригонометрию:

- `SinFunction` использует `TaylorSinCalculator`;
- `CosFunction` выражен через `sin`;
- `TanFunction`, `CotFunction`, `SecFunction`, `CscFunction` строятся через `sin` и `cos`.

### `function.log`

Здесь находится логарифмический слой:

- `LnFunction` использует `LnSeriesCalculator`;
- `LogNFunction` строит логарифм по основанию `a` через отношение `ln(x) / ln(a)`.

### `function.system`

`FunctionSystem` выбирает ветку по значению `x`:

- для `x <= 0` используется тригонометрическая формула;
- для `x > 0` используется логарифмическая формула.

### `stub`

`TableFunction` нужен для интеграционного тестирования. Он читает значения из CSV:

- если `x` совпал с точкой в таблице, возвращает точное значение;
- если `x` между двумя точками, делает линейную интерполяцию.

Это удобно для поэтапной интеграции системы: один модуль можно оставить реальным, а остальные временно заменить заглушками.

### `io`

Здесь лежит вся работа с выгрузками:

- `FunctionSampler` проходит по диапазону `[from, to]` с шагом;
- `CsvExporter` пишет файл формата `x,result`.

### `util`

Служебные классы:

- `DoubleComparator` проверяет `eps`, сравнивает числа и near-zero значения;
- `AngleNormalizer` уменьшает аргумент угла до удобного диапазона.

### `factory`

`FunctionFactory` собирает production-объекты в готовую цепочку зависимостей.

### `model`

- `MathFunction` - общий интерфейс всех функций;
- `SampledPoint` - точка для CSV-экспорта.

## 7. Как Построены Базовые Функции

### `sin(x)`

`sin(x)` реализован через ряд Тейлора:

```text
sin(x) = x - x^3/3! + x^5/5! - ...
```

Перед вычислением угол нормализуется, чтобы ряд сходился устойчивее.

### `ln(x)`

`ln(x)` реализован через численный ряд для положительных `x`. В коде учитывается область определения:

- `x <= 0` запрещен;
- `x = 1` обрабатывается как `0`.

## 8. Какие Производные Функции Построены На Их Основе

На основе базовых функций построены:

- `cos(x)`
- `tan(x)`
- `cot(x)`
- `sec(x)`
- `csc(x)`
- `log2(x)`
- `log3(x)`
- `log5(x)`

Важно для защиты: ни одна из этих функций не использует `Math.sin()` или `Math.log()` в production-коде.

## 9. Что Такое Табличные Заглушки И Зачем Они Нужны

Табличная заглушка - это временная реализация функции на основе заранее подготовленной таблицы значений. Она нужна для интеграционного тестирования по модулям.

Пример идеи:

1. Оставляем реальными тригонометрические функции.
2. Логарифмы временно подменяем табличными CSV-заглушками.
3. Проверяем, что тригонометрическая ветка системы правильно интегрируется.
4. Затем делаем наоборот для логарифмической ветки.

Это и есть основная идея лабораторной.

## 10. Выбранная Стратегия Интеграции

Использована поэтапная интеграция сверху вниз с заглушками:

1. Интеграция тригонометрической ветки с заглушками логарифмов.
2. Интеграция логарифмической ветки с заглушками тригонометрии.
3. Интеграция полной системы на реальных реализациях.
4. Интеграция экспорта в CSV.

Почему это удобно:

- хорошо видно, как система собирается из модулей;
- легко объяснить, почему именно такие integration tests были написаны;
- удобно показывать преподавателю пошаговую сборку системы.

## 16. Как Построить Графики Через Python

В проект добавлен скрипт:

```text
scripts/plot_graphs.py
```

Он читает CSV и строит PNG-графики.

Установка зависимостей:

```bash
python3 -m pip install -r requirements.txt
```

Запуск:

```bash
python3 scripts/plot_graphs.py --input-dir output --output-dir plots
```

Что делает скрипт:

- читает CSV формата `x,result`;
- ищет известные файлы: `sin.csv`, `ln.csv`, `log2.csv`, `log3.csv`, `log5.csv`, `cos.csv`, `tan.csv`, `cot.csv`, `sec.csv`, `csc.csv`, `system_left.csv`, `system_right.csv`, `system.csv`;
- если файл отсутствует, пишет понятное сообщение и идет дальше;
- строит отдельный PNG на каждый найденный CSV;
- для системы строит совмещенный график, где на одной картинке показаны аналитическая и аппроксимационная кривые;
- корректно обрабатывает `NaN` и `inf`, чтобы не рисовать некрасивые линии через разрывы.
